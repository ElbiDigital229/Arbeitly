\# Arbeitly – Candidate Journey (Paid Plan)

\---

\#\# 1\. Overview

The Paid Plan transforms Arbeitly into a \*\*hybrid job application platform\*\*:

\- Candidate can continue using all free features (self-service)  
\- Employees simultaneously apply on their behalf  
\- AI enhances and generates application materials

\---

\#\#\# Key Principle:

\> Paid Plan \= Free Plan \+ Managed Service Layer

\---

\#\# 2\. Entry Point

Candidate lands on:  
→ Pricing Page

Selects:  
→ Paid Plan

\---

\#\# 3\. Account Creation & Payment Flow

\---

\#\#\# 3.1 Signup

Fields:  
\- Full Name  
\- Email  
\- Password

\---

\#\#\# 3.2 Payment Step

User is redirected to:  
→ Payment Page

\- Card details

\---

\#\#\# 3.3 Payment Success

System:  
\- Creates candidate account  
\- Assigns selected plan  
\- Records transaction  
\- Marks user as \*\*Paid\*\*

\#\# 8\. Module 2: Files (Unified Storage)

\---

\#\#\# 8.1 Purpose

Single source of truth for all assets

\---

\#\#\# 8.2 File Types

\- Candidate-created CVs  
\- AI-generated CVs  
\- Employee-created CVs  
\- Job-specific CVs  
\- Cover letters (optional future)

\---

  \- System

→ Self-service \+ managed service working together

\- AI improves quality    
\- Everything is visible in one place
